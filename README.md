## 为什么要选住宅IP？

这事儿得从IP的类型说起。

平常我们用的VPS，大多是机房IP，也就是数据中心IP。这种IP虽然便宜、速度快，但有个致命问题——平台知道你不是真实用户。像TikTok、Facebook、Instagram这些社交平台，它们的风控系统早就把全球主要数据中心的IP段摸得一清二楚。

你用机房IP去注册账号、发视频，平台一看："嘿，这哥们儿用的是服务器IP，十有八九是搞批量操作的。"然后你的账号就进了小黑屋。

住宅IP就不一样了。它们来自真实的家庭宽带，就像你坐在美国某个家庭的客厅里用Wi-Fi上网一样。平台根本分辨不出来你是真用户还是在用VPS，自然也就不会限制你。

而双ISP住宅IP更厉害，它有两个不同运营商的线路同时接入，比如ATT + Verizon，或者NTT + SoftBank。这样不仅稳定性更好，而且在平台眼里，你这个"家庭用户"的网络环境更真实、更高端。

## 不适合什么人用？

也要说说IPRaft的局限性。

### 追求极致性能的用户

IPRaft的硬件配置是够用的水平，但不是顶尖的。

如果你要跑复杂的AI模型训练、高负载的数据库、大型网站，IPRaft的配置会吃力。这种场景建议选择专业的云服务器商家，比如AWS、Google Cloud，虽然贵但性能确实强。

IPRaft的线路没有做CN2、GIA这种中国优化，到国内的速度和延迟表现一般。

如果你的业务主要面向国内用户（比如建站、做国内API接口），IPRaft不是好选择。去选那些主打中国优化线路的商家更合适。
